# 개인 개발자 포트폴리오 디자인

## 목표

기존 `index.html`을 채용 담당자와 개발자 동료가 함께 볼 수 있는 개인 포트폴리오로 바꾼다. 방문자가 짧은 시간 안에 개발자의 정체성, 기술, 대표 프로젝트, 연락 방법을 파악할 수 있어야 한다.

첫 버전은 GitHub Pages에서 별도 빌드 없이 공개할 수 있는 정적 원페이지 사이트로 만든다. 실제 소개 내용이 아직 준비되지 않은 부분에는 교체하기 쉬운 예시 문구를 사용한다.

## 기술과 파일 구성

- 기존 `index.html`을 GitHub Pages 진입점으로 유지한다.
- HTML, CSS, 필요한 JavaScript를 `index.html` 한 파일에 작성한다.
- 프레임워크, 패키지 관리자, 빌드 도구를 추가하지 않는다.
- 외부 데이터 요청이나 서버 기능은 사용하지 않는다.
- 비교 목업이 저장되는 `.superpowers/` 디렉터리는 Git에서 제외한다.

단일 파일 방식은 현재 프로젝트 규모에 맞고, HTML과 CSS를 학습하며 직접 수정하기 쉽다. 콘텐츠나 기능이 크게 늘어나면 그때 스타일과 스크립트를 별도 파일로 분리한다.

## 정보 구조

페이지는 위에서 아래로 다음 순서로 구성한다.

1. **내비게이션**
   - 개발자 이름 또는 이니셜을 브랜드로 표시한다.
   - 소개, 기술, 프로젝트, 연락처로 이동하는 내부 링크를 제공한다.

2. **히어로**
   - 개발자 역할과 한 줄 소개를 가장 먼저 보여준다.
   - 짧은 보조 설명으로 일하는 관점과 관심사를 전달한다.
   - 프로젝트 섹션과 GitHub로 이동하는 주요 행동 버튼을 둔다.

3. **소개**
   - 개발자로서 중요하게 생각하는 점과 현재의 성장 방향을 짧게 설명한다.

4. **기술**
   - 현재 사용할 수 있는 기술을 간결한 태그 또는 목록으로 보여준다.
   - 숙련도를 수치나 막대로 과장하지 않는다.

5. **프로젝트**
   - 대표 프로젝트 2~3개를 카드로 표시한다.
   - 각 카드에는 프로젝트명, 해결한 문제나 목적, 사용 기술, 관련 링크를 넣는다.
   - 링크가 준비되지 않은 예시 프로젝트는 클릭 가능한 것처럼 표현하지 않고 준비 중임을 표시한다.

6. **연락처**
   - 이메일과 GitHub 등 실제로 공개할 연락 수단을 제공한다.

7. **푸터**
   - 이름과 현재 연도를 간결하게 표시한다.

## 시각 디자인

승인된 방향은 균형 잡힌 원페이지 레이아웃과 클린 모던 스타일이다.

- 밝은 중성 배경과 짙은 본문 글자를 사용한다.
- 파란색을 링크, 주요 버튼, 작은 강조 요소에만 사용한다.
- 넉넉한 여백과 명확한 글자 크기 차이로 정보 위계를 만든다.
- 프로젝트는 단순한 카드로 구분하며 장식은 최소화한다.
- 데스크톱에서는 콘텐츠 폭을 제한해 읽기 쉽게 만들고, 모바일에서는 모든 섹션과 카드가 한 열로 배치되게 한다.
- 시스템 글꼴을 사용해 외부 글꼴 다운로드 없이 빠르게 표시한다.

## 상호작용

- 내비게이션과 히어로 버튼으로 페이지 내부 섹션에 부드럽게 이동한다.
- 프로젝트 카드와 링크에는 짧은 호버 및 포커스 피드백을 제공한다.
- 과한 등장 애니메이션이나 스크롤 효과는 사용하지 않는다.
- 사용자가 움직임 감소를 설정한 경우 부드러운 스크롤과 전환 효과를 최소화한다.

## 접근성과 내구성

- `header`, `nav`, `main`, `section`, `footer` 등 의미에 맞는 HTML 요소를 사용한다.
- 제목 단계가 순서대로 이어지도록 구성한다.
- 모든 링크와 버튼은 키보드로 접근할 수 있고 명확한 포커스 표시를 갖는다.
- 색상만으로 상태나 의미를 전달하지 않는다.
- 모바일 화면에서도 글자와 터치 영역이 지나치게 작아지지 않게 한다.
- 외부 링크는 실제 주소가 정해진 경우에만 활성화한다.

## 검증

구현 후 다음 항목을 확인한다.

- HTML 문법과 문서 구조
- 내비게이션 내부 링크의 대상
- 데스크톱 및 모바일 레이아웃
- 키보드 탐색과 포커스 표시
- 움직임 감소 설정
- 준비되지 않은 링크가 잘못된 페이지로 이동하지 않는지 여부
- GitHub Pages에서 정적 파일로 정상 표시되는지 여부
